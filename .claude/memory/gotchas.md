# Gotchas

Durable landmines. Read before editing or shipping.

## Codegen / tooling

- **Regenerate after model/DB/provider changes:** `dart run build_runner build
  --delete-conflicting-outputs`. `*.g.dart` / `*.freezed.dart` are
  analyzer-excluded — never hand-edit them; change the source and re-run.
- **Freezed 3** data classes are `abstract class X with _$X` (not `@freezed
  class`). Use `@JsonSerializable(explicitToJson: true)` semantics for nested
  freezed models in backup.
- **drift** uses `@DataClassName('…Row')` so generated row classes don't collide
  with the domain models of the same name.
- Run bare **`flutter analyze`** (whole project, covers `test/`) before pushing.

## Riverpod 3

- `AsyncValue.valueOrNull` is **not available** here — use `.asData?.value`
  (or `.value`). Using `valueOrNull` fails analysis. (Hit during the offline
  catalog work.)
- `dashboardVehicleProvider` must stay derived from `vehiclesProvider`, so a
  single `ref.invalidate(vehiclesProvider)` (e.g. after adding a vehicle)
  refreshes every vehicle-scoped screen. Don't make it query the repo directly.

## Vehicle catalog

- **Catalog is OFFLINE — do NOT reintroduce CarQuery.** CarQuery API is dead
  (HTTP 502 + support portal "Account Suspended"); it caused the old
  "catalogo non raggiungibile" / infinite-spinner bug. Source of truth is the
  bundled asset `assets/catalog/catalog.json` (`OfflineCatalog`). To extend the
  catalog, edit that JSON. No free API reliably gives EU L/100km + tank specs.
- `SpecSource` is `{ catalog, manual }` (was `carquery`); persisted by `.name`
  with a `manual` fallback for unknown values.

## Domain rules

- **Consumption** is full-to-full (partial fills accumulate); intervals missing
  liters render as "—", never estimated.
- **Reminder status** is derived by `ReminderEvaluator` (today + max odometer),
  never stored; recurrence anchors to completion.
- **Category `kind`** (fuel|expense) discriminates the shared table — filter by
  kind in every picker/query.

## Build / Android

- Use **`file_selector`, not `file_picker`** — file_picker broke the Android
  release build.
- App icon is generated by `flutter_launcher_icons` from `assets/icon/icon.png`
  + `icon_foreground.png` (adaptive bg `#00695C`). Regenerate with
  `dart run flutter_launcher_icons` after changing the source PNGs.
- Release APK is debug-signed unless `ANDROID_KEYSTORE_*` secrets are set.
- **In-app updater over-install requires matching signatures.** A release APK
  signed with the release keystore installs over a release-keystore build and
  preserves SQLite data; it will not install over a local debug-signed build
  without uninstalling first.
- **Station detection** added `ACCESS_COARSE/FINE_LOCATION` to the manifest
  (`geolocator`, when-in-use, requested on demand at fill-up time). OCR is
  on-device via `google_mlkit_text_recognition` (latin model bundled — no Play
  Services download, works offline). Online station lookup uses the Overpass
  API (`http`), is **opt-in only** (explicit "Cerca online"), and degrades to
  empty on any network error — never throws. Receipt parsing is best-effort:
  `ReceiptParser` pre-fills what it can and leaves the rest for manual entry.
- **R8 release-only failure (ML Kit):** the **release** build runs R8 (because
  core-library desugaring is on for flutter_local_notifications). R8 does a
  whole-program check and fails with `Missing class
  com.google.mlkit.vision.text.{chinese,devanagari,japanese,korean}…` because we
  bundle only the Latin model. **Debug builds use D8 and skip this check**, so a
  debug APK build is NOT enough — it broke the **CD** while CI (debug) was green.
  Fixed with `-dontwarn` rules in `android/app/proguard-rules.pro` (wired via
  `proguardFiles` in the `release` buildType). CI `build-apk` now builds
  `--release` so R8 runs and catches this class of bug per-PR.

## Tests

- In-memory test DB is `makeTestDb()` (`test/helpers/test_db.dart`); override
  `appDatabaseProvider` (and `catalogRepositoryProvider` for the wizard) in
  widget tests.
